# Checklists — Jira Cloud Admin

## 1) Proyectos (Company vs Team)
- Confirmar tipo de proyecto y limitaciones
- Revisar configuración de roles/miembros
- Verificar esquema asociado (si aplica)

## 2) Permisos
- ¿Quién necesita qué? (create/edit/transition)
- Revisar esquema de permisos y grupos
- Evitar permisos globales innecesarios

## 3) Workflows
- Estados mínimos necesarios
- Transiciones con condiciones/validadores
- Post-funciones sólo si aportan valor

## 4) Campos y pantallas
- Evitar campos globales sin contexto
- Revisar screens por tipo de issue
- Marcar campos requeridos con criterio

## 5) Automatización
- Definir objetivo claro de la regla
- Probar en un proyecto sandbox
- Revisar límites y auditoría

## 6) JSM (Servicio)
- Tipos de solicitud y formularios
- SLAs definidos y medibles
- Colas útiles para operación

## 7) Reportes
- Filtros compartidos con permisos
- Dashboards por equipo
- Gadgets accionables
